# AlexBelog Game Request v3.3

Минималистичный статический сайт под **GitHub Pages** с интеграцией **Supabase** и автоподтягиванием примерной длительности из **HowLongToBeat**.

## Что нового
- более чистый и минималистичный визуал
- бесшовный фон без резких декоративных блоков
- иконка Twitch в шапке вместо обычной заглушки
- маленький значок **HLTB** рядом с полем названия
- автозаполнение поля **«Примерная длительность, часов»** при вводе названия игры
- быстрые ссылки на HLTB из очереди, слотов и карточки игры
- сохранена совместимость с GitHub Pages и Supabase

## Файлы
- `index.html` — интерфейс
- `styles.css` — оформление
- `app.js` — логика сайта, HLTB и админки
- `supabase.sql` — таблица, индексы и RLS policy

## Как развернуть на GitHub Pages

### 1. Создай репозиторий
Загрузи эти файлы в корень нового репозитория.

### 2. Включи GitHub Pages
В GitHub:
- `Settings`
- `Pages`
- `Deploy from a branch`
- Branch: `main`
- Folder: `/root`

### 3. Создай проект Supabase
В Supabase:
- создай новый проект
- открой `SQL Editor`
- вставь `supabase.sql`
- выполни скрипт

### 4. Создай пользователя для админки
В `Authentication`:
- добавь себе пользователя через email/password
- или используй magic link

### 5. Добавь URL сайта в Redirect URLs
В `Authentication` -> `URL Configuration` добавь:
- адрес GitHub Pages сайта
- и при желании локальный адрес для тестов

Пример:
- `https://USERNAME.github.io/REPO/`
- `http://127.0.0.1:5500/`

### 6. Вставь ключи в `app.js`
Найди строки:
- `PASTE_SUPABASE_URL`
- `PASTE_SUPABASE_ANON_KEY`

И замени их на значения из Supabase Project Settings.

## Как работает HLTB
- пользователь начинает вводить название игры
- сайт пробует найти наиболее подходящее совпадение в HLTB
- если матч найден, поле часов заполняется автоматически
- при желании значение можно вручную изменить

### Важно
Интеграция HLTB в статическом сайте опирается на браузерный клиент. Если HLTB меняет внутренний поиск или временно отвечает нестабильно, автоподстановка может перестать срабатывать — в этом случае остаётся ручной переход через кнопку/значок **HLTB**.

## Как работает сайт
- зритель отправляет заявку
- заявка попадает в таблицу `game_requests`
- публично видны все статусы, кроме `rejected`
- ты заходишь в админку и меняешь статус, приоритет и слот
- если Supabase не настроен, сайт всё равно работает локально в demo mode

## Быстрый локальный тест
Можно просто открыть сайт локально через любой static server. Например в VS Code через Live Server.
